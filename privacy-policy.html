Privacy Policy — Cemzo (Users App)

Effective date: October 24, 2025

App and contact

This privacy policy applies to the Cemzo mobile application (Android package com.cemzo.user, version 1.0.2) and describes what data the app collects, where it is stored, and why. For questions or data requests contact: contact@cemzo.com or call +91 79053 21205.

Permissions requested (Android)

The app's Android manifest (managed via app.json) requests the following runtime/declared permissions and uses them for the purposes noted below:

  • INTERNET & ACCESS_NETWORK_STATE — required for all network communication with Cemzo backend services (API base URL shown in source: https://api.example.com in api/axiosInstance.ts).
  • CAMERA — used to capture photos for profile images and for AR preview features (see app/screens/edit-profile.tsx and app/screens/ar-placement.tsx).
  • READ_EXTERNAL_STORAGE & WRITE_EXTERNAL_STORAGE — requested to allow selecting, saving, and uploading images/files from device storage (image picker and AR model uploads).
Only request permissions at runtime where the UI requires them (the codebase contains permission request placeholders near AR and profile image flows).

Data collected & where it is stored

Below are concrete, discoverable storage locations and network endpoints used by the app:

  • Authentication tokens and sensitive data — stored in Expo SecureStore via storage/secureStoreHelper.ts (keys such as auth_token, refresh_token). Tokens are attached to API requests by api/axiosInstance.ts.
  • Cached and non-sensitive data — stored in AsyncStorage via storage/asyncStorageHelper.ts (product lists, categories, user preferences, search history).
  • Structured offline data — cart items, wishlist, orders and in-app notifications are stored in the local SQLite database via database/sqliteHelper.ts.
  • Network backend — all network requests go to the configured API (https://api.example.com as seen in api/axiosInstance.ts). This is where data is transmitted and processed by Cemzo services.
  • Push notifications — the project includes expo-notifications in dependencies; notification delivery and related metadata are used to show order status, promotions and system messages and are persisted locally in SQLite (notifications table).

How we use data

  • Authenticate and authorize users (login tokens).
  • Enable shopping features: cart, orders, checkout and order history.
  • Support profile management and images (camera + storage).
  • Provide AR previews (requires camera and local image access).
  • Deliver push notifications and in-app messages.

Third-party sharing & external services

Data is sent to Cemzo backend services at the configured API host (see api/axiosInstance.ts). The codebase does not include explicit third-party analytics SDKs in dependencies (no analytics packages were found), but standard platform services (e.g. app store or payment processors used by backend) may process data outside the app. Any third-party processors are controlled by Cemzo backend policies.

User controls & data deletion

From the app you can:

  • Update and delete profile details via the profile screens (see app/(tabs)/profile.tsx and app/screens/edit-profile.tsx).
  • Clear local caches using app settings (AsyncStorage and SQLite rows can be cleared; secure tokens can be removed via SecureStore helper).
  • Request full data deletion or export by contacting contact@cemzo.com. Provide the email associated with the account to help locate data.

Security

Sensitive tokens are stored using Expo SecureStore (storage/secureStoreHelper.ts). Non-sensitive caches use AsyncStorage and structured local data uses SQLite. Network traffic should be sent over HTTPS; the axios instance in api/axiosInstance.ts attaches the Bearer token from SecureStore to API requests.

Children

The app is not intended for children. If you believe we have collected a child's data, contact us at contact@cemzo.com.

Changes to this policy

We may update this privacy policy to reflect changes in the app or legal requirements. The effective date at the top will be updated when changes are published.

Copy for Play Console

Cemzo collects user profile information (name, email, phone), authentication tokens stored securely on device, local app data (cart, wishlist, orders) stored in SQLite, and cached non-sensitive items in AsyncStorage. The app accesses the camera to capture profile photos and enable AR preview features, and requests storage access to select and save images.
